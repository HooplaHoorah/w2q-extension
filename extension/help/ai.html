Enable on-device AI (Gemini Nano) — optional

Web‑to‑Quest fully works without AI. These steps only enable the extra features that light up the Explain this step (AI) and Generate printable variants (AI) buttons.

  1. Install Chrome Canary (64‑bit)
  2. Create a fresh profile (don’t sign in yet)
  3. Enable the flags and relaunch Chrome:
    #prompt-api-for-gemini-nano
    #prompt-api-for-gemini-nano-multimodal-input (optional)
    #optimization-guide-on-device-model (if available)
    #enable-experimental-web-platform-features (some builds need this)
  4. Load the extension, open the side panel, then from the panel console run:
    await window.ai.canCreateTextSession()

If you see "ready" — all AI buttons enable automatically. If you see "after-download", leave Chrome open on Wi‑Fi. If it says "missing" on this profile/build, the API isn’t exposed yet (that’s ok — everything still works without AI).
